URGENT MOBILE RESPONSIVE FIX

The mobile view is broken. On iPhone, half the page is cut off to the right. The site is wider than the phone screen. It was working before, so a recent layout change likely caused horizontal overflow.

Fix the mobile layout so nothing is cut off.

Main problem:
Some container, feed row, filter row, composer, or card layout is using a fixed desktop width and forcing the whole page wider than the viewport.

Required fixes:

1. Add global overflow protection:

html,
body,
#root {
width: 100%;
max-width: 100%;
overflow-x: hidden;
}

* {
  box-sizing: border-box;
  }

2. On mobile, remove all fixed desktop widths.

Search for widths like:

* width: 1200px
* min-width: 900px
* min-width: 1000px
* left sidebar width being included
* grid-template-columns forcing desktop layout

Replace mobile widths with:

width: 100%;
max-width: 100%;
