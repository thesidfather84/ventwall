TASK 1 — ADMIN PASSWORD PROTECTION

The Admin section must be password protected.

Requirements:
- Nobody should access Admin by just clicking the sidebar.
- When clicking Admin, show a password screen first.
- Use a simple admin password from an environment variable:
  ADMIN_PASSWORD
- Do not hardcode the password in the frontend.
- If no ADMIN_PASSWORD exists, show:
  “Admin password not configured.”
- After correct password, allow access to Admin dashboard.
- Store admin session temporarily in browser sessionStorage, not localStorage.
- Add logout button inside Admin.
- Admin dashboard should include:
  - reported posts
  - moderation flags
  - remove post
  - ban user/session
  - search posts
